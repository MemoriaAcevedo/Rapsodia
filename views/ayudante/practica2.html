Recomendaciones
  1. Entregar ejemplos al alumno sobre el funcionamiento del software.
  2. Indicar las reglas de negocio que las funcionalidades deben cumplir.
  3. Verificar nombres representativos para las pruebas y su inicio con la palabra should (debería).
  4. Verificar las 3 etapas al momento de confeccionar una prueba unitaria: Preparación, Acto, Afirmación.
  5. Verificar el cumplimiento de las características de una prueba unitaria: Atómica, Independiente, Inicua, Rápida, Informativa.
  6. Verificar las pruebas automatizadas implementadas haciendo uso Test Driven Development, permitan al alumno implementar las reglas de negocio y comportamiento correcto de las funcionalidades.
  7. Verificar que el código de las funcionalidades implementadas cumplan las reglas de negocio
  8. Verificar aspectos de refactorización en el código y pruebas unitarias.
