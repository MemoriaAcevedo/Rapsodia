Enunciado

El alumno debe crear casos de prueba funcionales utilizando las historias de usuario y pruebas de aceptación del módulo bancario. Una vez confeccionados los casos de prueba, debe proceder a identificar incidencias dentro del módulo y dejar evidencia de su existencia. Se recomienda previamente que haya leído el material teórico y tomar en cuenta los tips que se entregan en cada tópico.

Instrucciones

1. Dirigirse a la sección Historias de usuario para tener contexto sobre las funcionalidades del sistema con las historias de usuario y pruebas de aceptación del módulo bancario.

2. Dirigirse a la sección Pauta de evaluación, para observar el listado de incidencias que posee el módulo bancario dañado e inspeccionar el módulo bancario correcto ingresando con su rut y contraseña de la plataforma.

3. Dirigirse a la sección Módulo bancario dañado, para explorar el software bancario dañado identificandose con su rut y contraseña de la plataforma.

4. Verificar las pruebas funcionales creadas por los alumnos, en la sección Gestión de pruebas funcionales.

5. Verificar las incidencias identificadas por los alumnos, en la sección Gestión de incidencias.

6. Resolver dudas al alumno sobre el funcionamiento del software, en la sección Canal de comunicación

Enunciado

El alumno debe reparar el código del módulo bancario simplificado para la práctica utilizando el lenguaje de programación Java en la plataforma Codenvy y utilizando como base el reporte de incidencias y las conversaciones con el corrector (Product Owner), confeccionará pruebas unitarias automatizadas a los métodos y clases reparadas con JUnit. Además haciendo uso de la técnica Test Driven Development deberá implementar las funcionalidades faltantes de módulo bancario las cuales son Retiro y Transferencia.

Instrucciones

1. Dirigirse a la sección Gestión reparación módulo bancario. En esta se especifican las instrucciones necesarias para acceder a los ambientes de trabajo de los alumnos.

2. Configurar para tener acceso a la práctica de los alumnos, en la sección Instrucciones de configuración correctores.

3. Ingresar en el ambiente de trabajo del alumno en la plataforma Codenvy.

4. Verificar el avance en la reparación de los métodos y confección de pruebas unitarias automatizadas.

5. Comunicarse con el alumno para resolver dudas, en la sección Canal de comunicación
