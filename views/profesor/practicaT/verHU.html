Historias de usuario

A continuación se presentan las historias de usuario y pruebas de aceptación del módulo bancario

Recomendaciones
  1. Leer detenidamente cada historia de usuario y pruebas de aceptación
  2. Resolver todas las dudas que surgan respecto a la documentación (Historias de usuario/Pruebas de aceptación) y funcionamiento del software al alumno.
  3. Mantener continua comunicación con el alumno para explicarle el contexto del negocio.
  4. Indicar las reglas de negocio que las funcionalidades del software deben cumplir al alumno.
  5. Entregar ejemplos a los alumnos sobre el funcionamiento del software.
  6. Mencionar la importancia de las migas de pan, botón de atrás, ventanas de confirmación y mensajes de retroalimentación al usuario.

HU-3


Prueba de aceptación


Reglas

  1. No se permite depositar montos igual o menores que 0
  2. El sistema soporta almacenar solamente 1000000
  3. Manejar el valor máximo soportado por el campo monto que es de 2147483647 y -2147483647
  4. Notificar al usuario mediante un mensaje sobre el formato del monto a depositar. Ejemplo 10000
  5. Notificar al usuario mediante un mensaje sobre el estado de su depósito.
  6. Notificar al usuario mediante un mensaje sobre las restricciones del sistema en el caso que no las cumpla.

Prueba de aceptación


Reglas

  1. Validar el formato del rut, Formato correcto: 9214907-2
  2. Entregar un ejemplo al usuario sobre el formato correcto del rut

  8. Notificar al usuario mediante un mensaje sobre el formato del monto a transferir. Ejemplo 10000
  9. Notificar al usuario mediante un mensaje sobre el estado de su transferencia.
  10. Notificar al usuario mediante un mensaje sobre las restricciones del sistema en el caso que no las cumpla.
