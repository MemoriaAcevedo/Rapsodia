Instrucciones de configuración

Creación de cuenta


1. Ingrese a la página oficial de codenvy, y diríjase a la opción "login".



2. Seleccione la opción "Create One" e registrese con el e-mail con el cual se registró en Rapsodia. En el caso que usted posea una cuenta en Google+ con el e-mail de Rapsodia, seleccione el ícono de "Google+".






3. Una vez registrado será notificado a su e-mail para confirmar su cuenta en codenvy".



4. Para finalizar el registro de la cuenta, debe ingresar los datos solicitados en el formulario y seleccionar la opción "Get Started".



5. Página de inicio cuenta de usuario.

Creación de ambiente de trabajo


1. Como pre-requisito debe poseer y estar autenticado en su cuenta en github. A continuación se realizará una copia del repositorio de práctica del módulo bancario donde se encuentra el código a reparar (URL Repositorio: https://github.com/sebastiancevedo/practicaRapsodia), a un repositorio propio. Para realizar esto seleccione la opción "Fork".


2. Seleccione el contexto de su cuenta de github donde desea copiar el repositorio.


3. Una vez copiado el repositorio a nuesta cuenta de github, se procede a configurar el ambiente de trabajo. Para comenzar le indicamos a la plataforma que utilizaremos código que proviene desde nuestra cuenta de github seleccionando así las opciones "Import from existing location" y "Github". Ya seleccionadas estas opciones debe conectar la plataforma con github utilizando la opción "Connect Your Github Account"



4. Una vez que la plataforma se haya conectado con github, debe indicar el contexto de su repositorio donde copió el código de práctica y seleccionar el repositorio con el código a utilizar en la plataforma.



5. Seleccione el recipiente el cual posee las características necesarias para la actividad, para este caso debe seleccionar "Java".



6. Seleccione la pila de librerias necesarias para el recipiente, para este caso debe seleccionar "Java".



7. Especifique el nombre del ambiente de trabajo, con el siguiente formato (rut del usuario)p(número de práctica) Ejemplo: 18486956-kp1. Además especifique la cantidad de RAM para el ambiente con 2GB de RAM



8. Finalmente indique el nombre de su proyecto, este debe poseer el mismo nombre que su repositorio en github, y seleccione la opción "Create".



9. A continuación para que su corrector pueda llevar un siguimiento de su trabajo, este será invitado a colaborar dentro del ambiente de trabajo creado, para esto diríjase a "Workspaces" y seleccione su ambiente de trabajo.



10. Dentro de la configuración del ambiente, dirijirse a "Share" y "Add Developer".



11. Indique el email del corrector y al final de este finalizar con punto y coma para que se active la opción "Share". Ejemplo acevedoc.sebastian@gmail.com;.



12. Seleccione el ambiente de práctica para ingresar a este.



13. Ambiente de práctica.

Configuración de ambiente de trabajo


1. Ingrese a su ambiente de trabajo.


2. Seleccione "practicaRapsodia [console-java-simple]" para ver la organización de las carpetas del proyecto.


3. Seleccione el paquete "org.eclipse.che.examples" para ver las clases del proyecto.



4. Seleccione el paquete "com.codenvy.example.java" para ver las clases para confeccionar las pruebas.



5. Para guardar su progreso en su repositorio de github. En primera instancia debe indicar al ambiente de trabajo sus credenciales de github, para esto diríjase a Profile -> Preferences.



6. Seleccione Git -> Committer, y especifique su nombre y e-mail de su cuenta de github.



7. Para guardar el progreso en su repositorio, diríjase a "Git".



8. Seleccione "Add To Index..." para agregar los cambios realizados al repositorio de github



9. Seleccione "Commit..." para confirmar los archivos que serán registrados en el repositorio.



10. Ingrese un comentario para identificar el commit a realizar y presione "Commit".



11. Seleccione "Remotes..." y "Push", de esta forma se envía el progreso a su repositorio de github.



12. Seleccione "Push".



13. Push realizado con éxito a su repositorio.



15. A continuación se configurará el comando para ejecutar el proyecto. Para esto seleccione "Edit Commands..."



16. Cree un nuevo comando presionando el ícono ubicado al lado derecho del título Maven. Posteriormente ingrese los datos especificados en la imagen en la definición del comando.



17. Seleccione en la sección de comandos "build" y presione el botón especificado en la imagen. De esta manera las clases del proyecto serán compiladas y se ejecutarán las pruebas automatizadas implementadas.



18. Terminal del ambiente de trabajo donde puede visualizar las acciones ejecutados por el comando "Build" y sus respectivos resultados.



19. Seleccione en la sección de comandos "practicaRapsodia: run" y presione el botón especificado en la imagen. Con esto el proyecto será ejecutado.



20. Terminal con el resultado de la ejecución del proyecto.
